# 角色

Zed 组织的每个成员都会分配一个角色，该角色决定了他们可以访问和配置的内容。

## 角色类型 {#roles}

组织的每个成员都会被分配以下三种角色之一：

| 角色 | 描述 |
| ---------- | ------------------------------------------------------ |
| **所有者** | 完全控制权限，包括计费和所有权转移 |
| **管理员** | 完全控制权限，但无计费权限 |
| **成员** | 标准访问权限，无特权操作 |

### 所有者 {#role-owner}

所有者对组织拥有完全控制权限，包括：

- 邀请和移除成员
- 分配和更改成员角色
- 管理计费、支付方式和发票
- 配置数据共享策略
- 禁用 Zed 的协作功能
- 控制成员是否可以使用 Zed 托管的模型和 Zed 的编辑预测
- 将所有权转移给其他成员

### 管理员 {#role-admin}

管理员拥有与所有者相同的功能，但他们不能：

- 访问或修改计费设置
- 转移组织所有权

此角色适合处理日常成员访问权限的团队负责人或经理，而无需查看付款详细信息。

### 成员 {#role-member}

成员拥有对 Zed 的标准访问权限。他们无法访问计费或组织设置。

## 管理用户角色 {#managing-users}

所有者和管理员可以在 Zed 仪表板的"成员"页面中管理组织成员。

### 邀请成员 {#inviting-members}

1. 在成员页面，选择 **+ 邀请成员**。
2. 输入成员的公司邮箱地址并选择一个角色。
3. 被邀请者会收到一封包含加入说明的电子邮件。接受后，他们通过 GitHub 进行身份验证。

### 更改成员角色 {#changing-roles}

1. 在成员页面，找到该成员。您可以按角色筛选或按名称搜索。
2. 打开三点菜单并选择新角色。

### 移除成员 {#removing-members}

1. 在成员页面，找到该成员。
2. 选择 **移除** 并确认。

移除成员会撤销他们对组织设置和任何组织管理功能的访问权限。他们可以继续独立使用 Zed。
